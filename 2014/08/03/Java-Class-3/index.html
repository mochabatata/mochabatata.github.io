Part 7 Inheritance & Polymorphism

“For better Life”

继承

编译器不会在乎方法实际从哪个类找到的
类关系：
is-a：单向结构，适用于继承
has-a
如何知道子类能够继承哪些？
通过设置access level来控制：普遍；public可继承 private不可继承
private < default < protected < public
使用情况:
某个类会比其父类更具有特定意义时，考虑到程序，应该被多个相同基本类型所共享是，应该考虑继承。
意义：
避免了程序代码重复；定义了共同的协议
防止某个类被做出子类：

  1. 存取控制，非公有的类只能被同一个包的类做出子类
  2. final 不能被继承（需要安全，确保方法是你版本用final）
  3. 让类只拥有private的构造程序（constructor）

遵守覆盖的规则：方法
参数 和 返回值 不能改变
不能降低方法的存取权限：只能更开放。例：public 变 private 不可行

多态

Dog myDog = new Dog()；
在多态下，引用和对象是不同类型，Animal myDog = new Dog();
运用多态时，引用类型可以是实际对象的父类型！（is-a类型的父类都可以！）
参数和返回值也可以多态：
举例：让猫叫，让狗叫 猫狗多态之后，对于animal类来说，定义makenoise(animal a)，makenoise(d)狗叫，makenoise(c)猫叫
程序代码为使用多态参数，即为参数声明为父类型，运行时可传入子类！
意义：通过多态，可以编写出引进新子类是也不必修改程序。（不必修改处理实例作为参数的方程了）

重载

方法的名称相同，但是参数不同
！：与多态毫无关系

性质：具有扩展性，但是不满足多态合约
1.返回类型可以不同
2.不能只改变返回类型（参数必须不同）
3.可以更改存取权限
