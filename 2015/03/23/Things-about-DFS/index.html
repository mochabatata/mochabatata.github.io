Things about DFS

Application:

Input Data:
Linked list, binary tree, set and array which use recursion.
Objective:
Get one solution only by stepping into the deepest.

How to solve?

Step 1
Path:

  • number of path : don’t need to store the path
  • path detail : if need only one solution, use path[] to store them; if need all solutions, use two-dimensional array to store them.
    Step 2
    If need only one solution, get one, and then return; if need all solutions, extend until finish traversal.
    Step 3
    How to store necessory data?
    add the data to function’s parameters. Use global variables to save recursion stack.
    Step 4
    How to extend the status?
    For binary tree and graph, go to the next level; For others, make clear what data should be store in the status?
    Step 5
    How to judge repetition？
    If the structure is tree, the repetition does not exist. If DAG, it means there is a child repetition problem.
    Step 6
    End condition
    For tre,it is leaf node. For gragh, it is the out-degree == 0.
    Step 7
    Convergence
    Recursion after parent process - it means parent state does not depend on child state. The recursion codes is in the end(tail recursion). Convergence if reach the goal.
    If parent state depends on child state, convergence if reach the initial state.
    Step 8
    Speed up
  • pruning
  • cache
